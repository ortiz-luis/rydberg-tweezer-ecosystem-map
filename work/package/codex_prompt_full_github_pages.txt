You have access to my GitHub repository. I am giving you a ZIP package containing the data files for a curated global database of Rydberg neutral-atom optical tweezer research groups, companies, theory nodes, and adjacent ecosystem actors.

Please build and publish a modern static GitHub Pages web app.

First inspect the files in this package, especially:

`data/rydberg_tweezer_ecosystem_map_export_v2_1.geojson`

Use that GeoJSON as the source of truth for the web app.

Repository goal:
Create a fully static, GitHub Pages-ready website showing all entities from the GeoJSON on an interactive world map with searchable, filterable detail cards.

Data handling:
- Copy the GeoJSON into the app at:
  `public/data/rydberg_tweezer_ecosystem_map_export_v2_1.geojson`
- Optionally copy the CSV/XLSX into `public/data/` or `public/downloads/` for users to download.
- Load the app data from the GeoJSON path.
- Do not hardcode entities.
- Preserve all GeoJSON properties.
- Use defensive parsing because field names may vary.
- Handle missing fields gracefully.

Technical stack:
- React + Vite + TypeScript
- Tailwind CSS
- Leaflet + react-leaflet
- No backend
- No database server
- No authentication
- Fully static deployment on GitHub Pages

Core features:
- Full-screen world map.
- One marker per GeoJSON Point feature.
- Marker click opens a detail panel.
- Left sidebar with:
  - title: “Rydberg Tweezer Ecosystem Map”
  - subtitle: “A curated map of research groups, companies, and theory nodes working on neutral-atom Rydberg platforms and optical tweezer arrays.”
  - search input
  - filters for classification, entity type, country, species, and tags when available
  - visible count of filtered entities versus total entities
  - reset filters button
- Detail card showing useful metadata when available:
  - name
  - PI or lead
  - institution
  - city
  - country
  - classification
  - entity type
  - platform tags
  - species
  - Rydberg role
  - optical tweezers status
  - experimental/theory/mixed status
  - key papers
  - source links
  - notes
  - confidence or curation status
- Legend explaining marker colors and classification A/B/C.
- Loading, empty, and error states.

Design:
- Modern, scientific, elegant, serious.
- Dark high-contrast visual style.
- Responsive desktop/mobile layout.
- Map occupies most of the screen.
- Sidebar should feel like a polished data explorer.
- Clean rounded cards, subtle borders, readable typography.
- Avoid toy-like colors or unnecessary animation.

GitHub Pages:
- Configure `vite.config.ts` with the correct `base` for this repository.
- Add `.github/workflows/deploy.yml` to build and deploy to GitHub Pages.
- Ensure `npm run build` works.
- Ensure the GitHub Actions deployment workflow is valid.
- Add README instructions for:
  - local development
  - updating the GeoJSON data
  - building
  - deploying through GitHub Pages
  - curation disclaimer
